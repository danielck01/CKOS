# 05_UI_UX_REFERENCES

## Funcao da pasta

Guardar referencias visuais e UX brutas sem iniciar design system ou implementacao.

## O que pode entrar

Prints, links, imagens, moodboards, benchmarks de produto, referencias de motion e estudos visuais.

## O que nao pode entrar

Componentes React, UI final, CSS de producao, prototipo navegavel ou decisao visual canonica.

## Naming obrigatorio

`YYYYMMDD_source-tool_topic_owner_status.ext`

## Regra de saida

Gerar nota em `000_STUDY_NOTES/04_UI_UX_STUDY/` quando houver insight aplicavel.

## Quando mover para STUDY

Quando a referencia revelar padrao, risco, oportunidade ou requisito visual futuro.

## Quando arquivar

Quando for generica, sem origem, fora da linguagem CKOS ou duplicada.

## Risco principal

Antecipar UI/UX antes dos docs 25-31.

## Politica de PII

Remover dados de usuarios, clientes, mensagens privadas ou paineis reais.

## Politica de provenance

Registrar origem, data de captura, ferramenta e contexto visual.
