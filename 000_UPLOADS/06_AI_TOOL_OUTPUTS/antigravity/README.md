# antigravity

## Funcao da pasta

Guardar outputs brutos do Antigravity, especialmente pesquisa visual ou revisao auxiliar.

## O que pode entrar

Analises visuais, pesquisas, revisoes e outputs auxiliares.

## O que nao pode entrar

UI final, componente, codigo, decisao canonica ou implementacao.

## Naming obrigatorio

`YYYYMMDD_antigravity_topic_owner_status.ext`

## Regra de saida

Mover para study note quando houver insight verificavel.

## Quando mover para STUDY

Quando o material ajudar docs futuros, especialmente UI/UX, conectores ou pesquisa.

## Quando arquivar

Quando for visual generico, duplicado ou sem origem.

## Risco principal

Disparar design/implementacao antes dos docs 32-33.

## Politica de PII

Remover dados sensiveis em prints ou exemplos.

## Politica de provenance

Registrar tarefa, origem e contexto.
