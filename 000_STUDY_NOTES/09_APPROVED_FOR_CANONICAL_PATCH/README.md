# 09_APPROVED_FOR_CANONICAL_PATCH

## Funcao da pasta

Guardar patch candidates aprovados para elaboracao de patch plan.

## O que pode entrar

Candidates revisados por PMO e aprovados para planejamento de patch.

## O que nao pode entrar

Doc canonico final, arquivo bruto, estudo incompleto ou patch ja aplicado.

## Naming obrigatorio

`YYYYMMDD_source-tool_topic_owner_status.md`

## Regra de saida

Preparar patch plan separado, submeter Founder e QA, aplicar apenas se aprovado.

## Quando mover para CANONICAL

Nunca como movimento. Esta pasta nao e canonica.

## Quando arquivar

Quando approval expirar, decisao mudar ou patch for rejeitado.

## Risco principal

Confundir pronto para patch plan com aprovado para alterar doc oficial.

## Politica de PII

Nao manter PII bruta em candidate aprovado.

## Politica de provenance

Manter links para study notes, source manifests e decision logs.

## Regra dura

`09_APPROVED_FOR_CANONICAL_PATCH` nao e canonico. Significa apenas pronto para patch plan.
