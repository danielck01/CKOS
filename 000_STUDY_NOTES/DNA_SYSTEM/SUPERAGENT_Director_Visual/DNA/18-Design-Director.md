## 0. Doutrina

A maior fonte de aprendizado em design não é teoria — é **referência cuidadosamente curada e analisada**. Designer que olha 50 referências fortes por semana evolui mais que designer que lê 5 livros teóricos por ano. Olho se forma por exposição.

A diferença entre marca com identidade visual forte e marca sem é frequentemente diferença de **biblioteca de referência**. Marca que mantém moodboard vivo, atualizado, analisado tem repertório pra discutir decisão visual concretamente. Marca sem biblioteca debate cada decisão na base do "achismo".

Toda decisão deste documento parte de uma premissa: **referência sem análise é decoração**. Ter pasta cheia de prints de Pinterest não é biblioteca — é coleção. Biblioteca exige análise: o que essa peça ensina? Por que funciona? Que padrão se repete entre múltiplas refs?

A função deste arquivo é tornar a biblioteca de referência **um organismo vivo** que alimenta toda decisão criativa da marca. Não congelado. Não decorativo. Operacional.

## 2. Os 12 princípios fundadores

### Princípio 1 — Curadoria é análise, não coleção

Pasta cheia de prints sem análise é entulho visual. Biblioteca útil exige **análise por referência**:

- O que essa peça ensina?
- Por que funciona?
- Que padrão técnico está visível?
- O que poderia ser aplicado na nossa marca?

Sem análise, a referência não vira repertório — fica como decoração estática.

### Princípio 2 — Quantidade não vence qualidade

50 referências cuidadas + analisadas vencem 500 prints aleatórios. Marca que tenta ter "moodboard de 200 imagens" geralmente não consegue navegar a própria biblioteca quando precisa.

Padrão saudável: 30-80 refs aprovadas por marca, divididas em 5-7 categorias, com 1-2 linhas de análise por ref.

### Princípio 3 — Referência precisa contexto

Imagem solta no Pinterest perde 80% do valor. Ref útil tem:
- Fonte (de onde veio)
- Autor/fotógrafo/designer (quem fez)
- Data (quando foi feito)
- Contexto (pra quê serviu)
- Análise (o que ensina)

Sem contexto, ref vira inspiração genérica.

### Princípio 4 — Anti-referência é tão importante quanto referência

Identidade visual se constrói tanto por inclusão quanto por exclusão. Biblioteca de "o que NÃO queremos parecer" guia tanto quanto biblioteca de "o que queremos".

Toda marca deveria ter, no mínimo:
- 30-50 referências positivas (inclusão)
- 10-20 anti-referências (exclusão)

### Princípio 5 — Vision real > descrição textual

Referência visual armazenada **como imagem** vale mais que como descrição textual. IA que lê biblioteca via vision real (Claude Read, GPT-4V) extrai padrões visuais que descrição em texto perde.

Implicação: biblioteca digital que tem só links sem imagens armazenadas é frágil. Imagens precisam estar na pasta `materiais/3-referencias-visuais/` localmente, ou em pasta sincronizada (Drive, Notion) acessível.

### Princípio 6 — Grid como referência composta

Print de feed (Behance, Instagram, Pinterest) com vários slides numa imagem = grid. Cada região do grid é uma ref independente.

Ref útil: **uma imagem em grid bom pode valer 9 refs separadas**. Não force quantidade — anexe o que de fato representa.

### Princípio 7 — Curadoria contínua > big bang

Biblioteca útil não é montada uma vez no setup e esquecida. É alimentada **continuamente**:
- 5-15 refs novas por semana (R1 Brand Scout)
- Curadoria humana semanal (20 min)
- Refresh trimestral (retirar refs envelhecidas)
- Snapshot anual no Drive

Sem alimentação contínua, biblioteca envelhece em 18 meses.

### Princípio 8 — Diversidade de fontes evita endogamia

Marca que só consume referência de Pinterest acaba parecendo Pinterest. Marca que diversifica fontes (Are.na + Behance + revistas físicas + livros + galeria + arquitetura + cinema) constrói repertório único.

Fontes recomendadas:
- 30% Are.na (curatorial)
- 20% Pinterest (popular mas filtra)
- 15% revistas/livros físicos
- 15% sites institucionais (Pentagram, Wolff Olins, MoMA)
- 10% cinema/teatro/exposições
- 10% concorrentes diretos (calibração)

### Princípio 9 — Detecção de padrão é onde valor mora

5 referências individualmente boas não ensinam tanto quanto 5 referências analisadas em conjunto pra extrair padrão recorrente.

Análise de padrão:
- Que tipografia se repete?
- Que tipo de luz?
- Que paleta?
- Que tipo de composição?
- Que detail signature aparece em múltiplas?

Padrão recorrente = sistema implícito. Documentar = sistema explícito.

### Princípio 10 — Refresh evita estagnação

Estética muda. Tendências envelhecem. Biblioteca de 2020 não serve 2026. Refresh estruturado:

- **Trimestral:** retirar refs claramente envelhecidas
- **Semestral:** snapshot completo da biblioteca (arquivo histórico)
- **Anual:** revisão crítica — biblioteca ainda representa marca atual?

Marca que não faz refresh acaba presa em estética envelhecida sem perceber.

### Princípio 11 — Pacote temático > biblioteca única

Quando inicia projeto específico (campanha, lançamento, redesign), criar **pacote temático** focado:

- 15-30 refs específicas pro projeto
- Anti-refs específicas
- Notas de direção pro projeto

Pacote temático tem prioridade sobre biblioteca geral durante o projeto. Foco temporário.

### Princípio 12 — Biblioteca é viva, não monumento

Pior anti-padrão de biblioteca: criar sistema elaboradíssimo, abandonar em 6 meses. Biblioteca útil é **proporcional à frequência de uso**.

Princípio: biblioteca cresce naturalmente quando é consultada. Se ninguém consulta, está morta — não vale manter.

Marca pequena: biblioteca enxuta de 30-50 refs, refresh trimestral.
Marca grande: biblioteca ampla com sub-categorias, curadoria semanal.

## 3. Estrutura da biblioteca — 6 seções operacionais

### 3.1 — Seção A: Refs visuais aprovadas (curadas por humano)

Imagens que definem o sistema visual da marca. Vision real as analisa quando R2 vai gerar peça nova.

Cada referência aprovada tem:
- A imagem (anexada)
- Descrição (1-2 linhas explicando o que faz funcionar)
- Categoria (capa, slide interno, hero web, post social, outro)
- Tags (palavras-chave que ajudam R2 a buscar)
- Padrão observado (que técnica visível)

**Quantidade ideal:**
- Capas de carrossel: 5-15
- Slides internos: 10-20
- Hero shots: 5-10
- Posts editoriais: 10-20
- Mood boards (grids): 3-8

Total: ~50-100 refs visuais é uma biblioteca saudável.

### 3.2 — Seção B: Refs editoriais aprovadas (textos modelares)

Textos que **representam o tom certo**. Funcionam como calibração viva da `🗣️ Voice & Tone`.

Cada referência tem:
- O texto (cole inteiro ou link + summary)
- Tipo (caption, post longo, email, headline, CTA, etc.)
- Por que funciona (1-2 linhas analisando)
- Origem (próprio aprovado / outra marca admirada)

**Quantidade ideal:**
- Captions: 10-20
- Posts longos: 5-10
- Emails: 5-10
- Headlines: 10-30 (banco de inspiração)
- CTAs: 5-15

### 3.3 — Seção C: Refs comportamentais

Prints de DMs bem-feitas, comunicados de crise modelares, threads exemplares.

Cada referência:
- Print/captura (anonimizar dados sensíveis)
- Contexto (o que rolou)
- Por que funciona (análise)

### 3.4 — Seção D: Anti-referências

Listadas explicitamente — quanto mais específico, melhor o resultado.

**Anti-referências visuais:**
- Não queremos parecer minimalismo Apple genérico
- Não queremos parecer gradient roxo IA tech
- Não queremos parecer dashboard SaaS azul
- Não queremos parecer Bauhaus básico
- Não queremos parecer textura de papel rasgado

**Anti-referências editoriais:**
- Não queremos soar como guru de marketing
- Não queremos soar como agência tradicional
- Não queremos soar como tech bro
- Não queremos soar como coach motivacional

**Anti-referências comportamentais:**
- Não queremos agir como marca que apaga comentários críticos
- Não queremos agir como marca que dá BF de 70% e cobra inflado nos outros 11 meses
- Não queremos agir como marca que finge ser pessoa nas DMs

### 3.5 — Seção E: Pendentes (R1 alimenta, humano cura)

R1 (Brand Scout) coleta automaticamente inspirações relevantes pra marca e arquiva aqui.

**Como R1 coleta:**

| Fonte | O que busca |
|---|---|
| Pinterest (boards relacionados ao brand_aesthetic_anchor) | Imagens que casam com a estética |
| Are.na (channels relacionados) | Refs editoriais e visuais curadas |
| Behance (categorias relacionadas) | Projetos completos |
| Dribbble (tags relacionadas) | UI / web / brand identity |
| Instagram (handles definidos como referência) | Posts novos das marcas-âncora |
| Designspiration / Cargo / Fwa | Refs editorial-leaning |

**Critério de coleta:**
- Casa minimamente com brand_aesthetic_anchor
- Não viola anti-referências (Seção D)
- Tem mínimo de 1 dimensão claramente alinhada (paleta, tipografia, composição, tom)

R1 pode trazer 5-15 pendentes por semana. Curadoria semanal: ~20min.

**Decisão de cada pendente:**
- ✅ Aprovar → move pra Seção A/B/C
- 🗂️ Arquivar (boa, não pra agora) → "Banco de futuro"
- ❌ Rejeitar → apaga

### 3.6 — Seção F: Mapa competitivo

Concorrentes monitorados — não pra copiar, pra **diferenciar conscientemente**.

| Concorrente | Handle | Pontos fortes | Onde diferimos |
|---|---|---|---|
| [Nome 1] | @ | [...] | [...] |

**Snapshots periódicos:**
R1 cria snapshot mensal de cada concorrente (5-10 posts mais recentes, padrão visual evoluindo, pivot tonal).

**Análise comparativa trimestral:**
A cada 90 dias, R2 gera análise comparativa estruturada.

## 4. Vision real — análise de imagens via IA

Quando R2 vai usar a biblioteca pra alimentar prompt:

1. `ls materiais/3-referencias-visuais/` lista as imagens
2. `Read` em cada imagem (vision nativa Claude)
3. Análise estruturada por imagem:
   - Tipografia visível
   - Paleta dominante + accent
   - Composição predominante
   - Iluminação (se foto)
   - Detail signature
   - Mood

4. Compilação em **briefing visual decodificado** (300-500 palavras):

```
VISUAL BRIEF (decoded from references):

Color palette:
- Primary: [hex] ([descrição])
- Background dark: [hex]
- Background light: [hex]
- Accent secondary: [observado nas referências]

Typography style:
- Headlines: bold condensed sans-serif, weight 900, very tight kerning, uppercase, similar in feel to [referência tipográfica]
- Body: clean humanist sans-serif, weight 400-500, medium contrast, similar to [referência]
- Tag: monospaced or geometric mono, weight 600, uppercase

Mood and atmosphere:
- [extraído das refs aprovadas — 2-3 frases]

Composition style:
- [extraído das refs aprovadas — 2-3 frases]

Texture and finish:
- [extraído das refs aprovadas — 1-2 frases]

Detail signature / footer:
- [observado em múltiplas refs como elemento recorrente]

What to AVOID (from anti-references):
- [lista da Seção D]
```

Esse briefing entra como **bloco fixo em todos os prompts** (capa e slides internos) daquela run.

## 5. Detecção de grids e decomposição

Quando alguém anexa uma imagem que é **grid** (print do feed, página de Behance, moodboard com vários slides), R2 detecta e decompõe.

**Sinais de grid:**
- Regularidade espacial (linhas/colunas claras)
- Bordas repetidas
- Formato quadrado ou retangular subdividido
- Múltiplos retângulos similares lado a lado

**Como decompor:**
- Grid 3×3 = 9 exemplares
- Print de feed = 6+ exemplares
- Print de Behance = N exemplares conforme layout

**Análise:**
Cada exemplar do grid é observado como ref independente. Padrões recorrentes entre exemplares = marcadores de consistência (sistema visual).

**Implicação prática:**
**uma única imagem em grid bom pode bastar.** Não force quantidade — anexe o que de fato representa.

## 6. Anti-referências — tão importantes quanto referências

### 6.1 — Por que anti-ref é estratégica

Identidade se constrói por **inclusão + exclusão**. Sem exclusão clara, marca dilui em 5 anos.

A IA generativa, especificamente, gravita pro genérico por design. Sem anti-prompt explícito, devolve média do treinamento. Anti-referência documentada vira `avoid:` no image_brief.

### 6.2 — Especificidade da anti-ref

Anti-ref vaga ("design ruim") não filtra nada. Anti-ref específica filtra tudo.

✅ Específico:
- "Não queremos parecer marca de SaaS B2B 2018-2022 com gradient roxo-azul + foto stock de equipe diversa sorrindo + heading 'A solução completa para...'"
- "Não queremos soar como coach motivacional brasileiro com 'acredite no seu potencial' + emoji fogo + áudio inspirador no fundo"

❌ Vago:
- "Design moderno"
- "Tom institucional"

### 6.3 — Como capturar anti-ref

Pergunta direta no briefing: "Cite 3 marcas/estilos/peças que vocês definitivamente NÃO querem parecer. Por quê?"

Cada anti-ref vem com:
- Print/exemplo (quando possível)
- Razão de evitar
- Categoria (visual, verbal, comportamental)

## 7. Curadoria contínua via R1 (Brand Scout)

### 7.1 — Frequência de coleta

R1 roda 3x por semana (default seg, qua, sex às 10h). A cada execução:
- Coleta 5-15 candidatas
- Adiciona em Pendentes (Seção E)
- Marca tags e fonte

### 7.2 — Curadoria humana

Toda quarta-feira (ou outro ritual semanal), curador humano (designer, fundador, brand person) gasta 20-30 min:
- Revisa pendentes
- Aprova / arquiva / rejeita cada uma
- Adiciona análise às aprovadas
- Move pra Seção A/B/C correspondente

### 7.3 — Refresh trimestral

A cada 90 dias:
- Revisa Seção A inteira: ainda representa o sistema?
- Retira refs claramente envelhecidas
- Adiciona refs novas se sistema evoluiu
- Re-organiza categorias se necessário

### 7.4 — Snapshot anual

Uma vez por ano:
- Export completo da biblioteca pra Drive
- Versão arquivada da biblioteca
- Análise: como biblioteca mudou em 12 meses?

Snapshot serve como **memória institucional**. Biblioteca de 5 anos atrás conta a história visual da marca.

## 8. Curadoria BOA vs RUIM (casos canônicos)

### 8.1 — Curadoria BOA (estudar)

**Casos exemplares:**

- **swissmiss** (swissmiss.com) — Tina Roth Eisenberg cura desde 2005, diariamente. Consistência editorial inconfundível.
- **It's Nice That** (itsnicethat.com) — curadoria editorial premium em design contemporâneo.
- **Brand New** (UnderConsideration) — análise crítica de identidades visuais. Cada análise é estudo de caso.
- **swissted** (swissted.com) — Mike Joyce reimagina pôsteres punk em estética suíça. Curadoria autoral em projeto.
- **Pentagram Papers** (50+ edições desde 1975) — curadoria de tópicos específicos por designers do estúdio. Modelo de curadoria autoral.
- **Stack Magazines** (stackmagazines.com) — entrega mensal de revista independente curada. Curadoria como serviço.

**Características em comum:**
- Frequência consistente (diária, semanal, mensal)
- Voz autoral (curador identificável)
- Análise por item (não só compilação)
- Diversidade de fonte (não só uma plataforma)
- Critério explícito (o que entra e por quê)

### 8.2 — Curadoria RUIM (evitar)

**Anti-padrões saturados:**

- **Pasta de Pinterest sem análise** — coleção, não biblioteca
- **Moodboard de 500 imagens** — ninguém consegue navegar
- **Biblioteca abandonada** — atualização zero há 18 meses
- **Curadoria sem voz** — coleção genérica sem ângulo
- **Tudo de uma fonte** — endogamia visual
- **Sem anti-ref** — só inclusão, sem exclusão

**Sintomas em marcas brasileiras médias:**
- Designer abre Pinterest pra "inspiração" cada projeto
- Sem biblioteca de marca documentada
- Cada peça nasce do zero (sem repertório acumulado)
- Equipe nova não tem onde estudar estética da marca

## 9. Biblioteca que FUNCIONA vs NÃO FUNCIONA (em escala)

### 9.1 — FUNCIONA

- **Biblioteca enxuta + analisada** (30-80 refs com análise)
- **Refresh trimestral** (alimentação contínua)
- **Vision real acessível** (imagens armazenadas, não só links)
- **Anti-refs documentadas**
- **Pacotes temáticos** quando inicia projeto
- **Curadoria semanal** (ritual incorporado)
- **R1 alimentando pendentes**

### 9.2 — NÃO FUNCIONA

- **Biblioteca enorme abandonada** (500 refs sem análise, sem refresh)
- **Refresh aleatório** (atualização quando "der tempo")
- **Só links sem imagens** (frágil — links morrem)
- **Sem anti-refs** (só inclusão)
- **Sem pacotes temáticos** (cada projeto começa do zero)
- **Curadoria esporádica** (sem ritual)
- **Coleta sem critério** (qualquer coisa entra)

## 10. Pacotes temáticos — curadoria por projeto

Quando inicia projeto específico (campanha, lançamento, redesign), criar **pacote focado** acelera direção criativa.

### 10.1 — Estrutura do pacote

```
Pacote: [Nome do projeto] — Q[N] [Ano]

Refs visuais aplicáveis (da Seção A):
- A-04, A-12, A-23, A-31

Refs editoriais aplicáveis:
- B-08, B-15

Refs adicionais específicas (só pra esse pacote):
[anexar 5-10 refs específicas]

Anti-referências relevantes pra essa campanha:
- "Nada que pareça campanha de Black Friday"
- "Sem urgência fabricada"
- "Sem influencer posing"

Notas de direção pro projeto:
- [observação 1]
- [observação 2]

Status: ATIVO / ARQUIVADO
Data início: [data]
Data fim: [data]
```

### 10.2 — Como R2 usa o pacote

Quando há "Pacote temático ativo", R2 prioriza-o sobre a Library inteira durante o projeto. Foco temporário.

Pacote serve como **brief curado pra projeto específico** — concentra direção criativa.

### 10.3 — Arquivamento de pacote

Quando projeto termina:
- Status: ARQUIVADO
- Mantém na Library pra histórico
- Refs específicas que sobreviveram = candidatas a entrar na Seção A permanente

## 11. Refresh e arquivamento

### 11.1 — Tarefas por frequência

| Tarefa | Frequência |
|---|---|
| Triar pendentes | Semanal |
| Re-curar Seção A (visuais aprovadas) | Trimestral |
| Re-curar Seção B (editoriais aprovadas) | Trimestral |
| Adicionar casos novos à Seção C | Quando rolar caso bom |
| Re-curar Seção D (anti-refs) | Trimestral |
| Atualizar Seção F (mapa competitivo) | Mensal |
| Snapshot completo no Drive | Trimestral |
| Revisão estratégica completa | Anual |

### 11.2 — Banco de futuro

Refs que são boas mas não cabem no momento:
- Estética muito ousada quando marca está em fase conservadora
- Referência cujo tema não está em pauta
- Tendência emergente que talvez vire relevante

Banco mantém pra revisão futura. A cada refresh trimestral, banco é re-avaliado.
