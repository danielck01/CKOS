# Miriam Material Build Pack

## Atualizacao AI-first - 2026-05-27

Foi adicionada a pasta `MIRIAM_PERSONAL_BRANDING/` como runtime documental de Briefing Vivo para CKOS + Branddock.

A pasta antiga continua preservada como material fonte e historico. A nova camada organiza contexto, source manifest, briefing vivo, next best question, Cognik, Metacognik, nodes Branddock, pesquisa, policies, workflows, prompts e outputs apenas em modo draft/analysis.

Regra atual: antes de qualquer HTML, bio, tom de voz, calendario ou posicionamento final, coletar respostas reais da Miriam e gerar `SOURCE_EVIDENCE_MAP.md` + `MATRIZ_DE_RISCOS_ETICOS.md`.

Este pack organiza a construção do material HTML da Miriam sem pular etapas, sem transformar o trabalho em landing page genérica e sem violar os limites éticos da publicidade jurídica.

## Objetivo

Criar uma base documental para que Codex, Claude, Antigravity ou outro chat especializado consigam produzir, revisar e evoluir um material de estudo/apresentação sobre a Miriam, usando as 3 Deep Research como fonte principal.

O resultado esperado depois deste pack é um HTML premium, mobile-first, bg black, com linguagem fácil, diagramas, exercícios, prompts copiáveis para imagens futuras e estrutura didática.

## Fontes que devem ser anexadas no chat executor

1. `deep-research-report (3).md` — pesquisa de branding para advogada penal, OAB, riscos e posicionamento.
2. `deep-research-report (2)(1).md` — algoritmo 2026 com base mais robusta e citações.
3. `Deep Research_ O Algoritmo de 2026 para Autoridade Profissional no Instagram e TikTok(1).md` — relatório complementar de autoridade profissional.
4. Este pack completo.

## Ordem correta de uso

1. `01_PROBLEMA/` — entender a dor real antes de criar design.
2. `02_ANALISE/` — transformar pesquisa em tese estratégica.
3. `03_COGNIK/` — interpretar intenção, contexto, sinais e fontes.
4. `04_METACOGNIK/` — auditar riscos, ética, lacunas e inconsistências.
5. `05_CEO_AGENT/` — decidir escopo, contratar agentes simulados e aprovar plano.
6. `06_PMO_AUDITOR/` — revisar execução, custo, risco e qualidade.
7. `07_SKILLS/` — escolher capacidades usadas na tarefa.
8. `08_INSTRUCOES/` — regras operacionais para o chat executor.
9. `09_PROMPTS/` — prompts prontos para gerar análise, HTML, imagens e QA.
10. `10_OUTPUTS_HTML/` — arquitetura do HTML, componentes e critérios de aprovação.

## Regra central

Nenhum output deve prometer resultado jurídico, sugerir captação agressiva, explorar crime como espetáculo ou transformar a Miriam em personagem sensacionalista.

A tese correta é: autoridade penal com ética, sobriedade, clareza educativa e presença feminina firme.
