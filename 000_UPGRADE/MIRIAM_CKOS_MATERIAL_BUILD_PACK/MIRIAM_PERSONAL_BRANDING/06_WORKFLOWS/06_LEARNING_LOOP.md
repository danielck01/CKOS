# Learning Loop

Status: active_protocol

## Funcao

Registrar aprendizados do projeto para melhorar proximos briefings CKOS + Branddock.

## Eventos de aprendizado

```yaml
learning_event:
date:
trigger:
what_changed:
source:
confidence:
applies_to:
  - miriam_project
  - legal_branding_projects
  - ckos_briefing_runtime
do_not_generalize_to:
```

## Exemplos

- Pergunta que destravou decisao.
- Risco que apareceu cedo.
- Fonte que invalidou hipotese.
- Gate que evitou output prematuro.

## Regra

Aprendizado nao altera arquitetura canonica do CKOS sem patch separado.
