# MVP Funcional 100%

O MVP é funcional quando transforma qualquer intenção em uma entidade operacional rastreável.

## O que precisa funcionar

1. Receber intenção livre.
2. Classificar tipo de briefing.
3. Criar Context State.
4. Selecionar agentes simulados ou reais.
5. Selecionar skills.
6. Selecionar tools/conectores disponíveis.
7. Gerar perguntas adaptativas.
8. Criar output acionável.
9. Criar tarefa/projeto/proposta/documento.
10. Registrar aprovação se necessário.
11. Registrar feedback.
12. Atualizar aprendizado.

## O que pode ser simulado no MVP

- Superagents podem ser perfis/prompt configs.
- Background tasks podem ser jobs assíncronos simples.
- Tools podem começar com web, RAG, file upload e Apify.
- Conectores podem começar com manual upload + APIs prioritárias.
- Learning pode começar como tabela de feedback e eventos.

## O que não pode faltar

- Intent Router.
- Context State.
- Question Engine.
- Output Engine.
- Policy Checker.
- Learning Event Logger.

Sem isso, vira só chat com templates.
