# Policy — Quando n8n vira Código Próprio

## n8n é permitido para

- prototipar integrações;
- testar webhooks;
- conectar APIs;
- criar alertas;
- backoffice;
- sincronização;
- automações não críticas;
- geração de proposta inicial;
- suporte operacional.

## Migrar para código próprio quando

- envolve pagamento;
- envolve crédito de usuário;
- envolve dados sensíveis;
- roda muitas vezes por dia;
- afeta experiência principal do cliente;
- precisa de testes automatizados;
- exige controle fino de erro;
- exige versionamento forte;
- tem impacto jurídico, financeiro ou reputacional.
