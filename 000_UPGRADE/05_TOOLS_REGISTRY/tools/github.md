# Tool — github

## Função
Repositórios, issues, PRs e código.

## Quando acionar
Quando o briefing precisar de ação ou dado que não deve ser inventado por LLM.

## Inputs comuns
- Query
- Context State
- Project ID
- Credentials/Secrets
- Policy constraints

## Outputs comuns
- Dados coletados
- Status
- Erros
- Evidências
- Custos
- Próximas ações

## Policy
Registrar uso, custo, origem dos dados e impacto na decisão.
