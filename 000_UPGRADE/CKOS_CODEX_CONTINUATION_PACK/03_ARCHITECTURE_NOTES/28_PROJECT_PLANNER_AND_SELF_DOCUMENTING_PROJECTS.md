# 28 — Project Planner and Self-Documenting Projects

## Propósito

Cada projeto dentro do CKOS deve ser capaz de gerar sua própria documentação de arquitetura, plano, riscos, sprints e aprendizado.

## O que isso evita

- Founder escrevendo prompt gigante toda hora.
- Agentes sem contexto.
- Documentação desatualizada.
- Decisões sem registro.
- Projetos que começam sem arquitetura.

## Fluxo ideal

1. Usuário cria projeto.
2. Nick faz briefing inteligente.
3. Cognik cria context pack.
4. Metacognik detecta lacunas.
5. Planner Agent sugere arquitetura do projeto.
6. PMO Agent cria roadmap.
7. Cost Agent estima consumo.
8. ROI Agent cria hipóteses de retorno.
9. QA Agent cria critérios de aprovação.
10. Founder aprova.

## Modos de trabalho

### CK Work

Orquestra tarefas, aprovações, sprints e stakeholders.

### CK Code

Prepara escopo técnico, patches, QA e execução com Claude/Codex.

### CK Design

Prepara direção visual, prompts de imagem, motion e UI futura.

### CK Research

Pesquisa web, acadêmica, social e RAG.

## Regra

Isso deve ser documentado agora, mas implementado só depois dos docs de business, collectors, learning e UI/UX.
