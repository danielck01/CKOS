# Prompt para Antigravity — Revisão Paralela sem Implementação

Atue como `Antigravity UI Systems Researcher`, mas sem construir UI.

Sua missão é estudar referências visuais e transformar em critérios documentais para a futura arquitetura UI/UX do CKOS.

## Proibido

- Não criar tela.
- Não criar componente.
- Não criar CSS.
- Não criar React.
- Não criar protótipo.
- Não alterar docs Runtime/Product/Business.

## Permitido

- Criar nota de pesquisa visual.
- Criar taxonomia de componentes.
- Criar lista de motion patterns.
- Criar vocabulário visual.
- Criar prompts para Visual Director.
- Criar critérios de UI futura.

## Output esperado

Criar:

`UI_RESEARCH_NOTES_FOR_FUTURE_DOC_30.md`

Com:

1. Linguagem visual CKOS.
2. Componentes observados nas referências.
3. Motion patterns úteis.
4. O que NÃO copiar.
5. Como traduzir para CKOS.
6. Componentes futuros prováveis.
7. Critérios para Project Dashboard.
8. Critérios para Command Center.
9. Critérios para Node Canvas.
10. Critérios para Learning Mode.
11. Critérios para CKStore.
12. Prompts de direção visual para gerar imagens depois.

## Tese visual

CKOS deve parecer um centro de comando cognitivo, não um app genérico de produtividade.

Fundo preto, glassmorphism premium, bento grid, cards grandes arredondados, movimento suave, widgets vivos, commandbar como centro de ação e interface que mostra o sistema pensando sem virar ruído.
