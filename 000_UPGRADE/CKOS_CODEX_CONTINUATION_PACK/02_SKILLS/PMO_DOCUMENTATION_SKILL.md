# PMO Documentation Skill

## Função

Garantir que qualquer IA trabalhando no CKOS siga ordem, escopo, naming, dependências, gates e relatórios.

## Quando usar

- Antes de criar documento novo.
- Antes de aplicar patch.
- Antes de renomear arquivo.
- Antes de atualizar versões.
- Antes de seguir para outro doc.

## Checklist

1. Qual documento está em escopo?
2. Qual versão anterior?
3. Qual versão nova?
4. Qual dependência será afetada?
5. Existe patch registrado?
6. Existe gate aberto?
7. O Founder precisa aprovar?
8. Isso cria risco de entropia?
9. Isso antecipa UI/UX indevidamente?
10. O relatório final cita tudo que foi alterado?

## Output padrão

- Arquivos alterados.
- Versões atualizadas.
- Patches aplicados.
- Patches pendentes.
- Riscos.
- Próximo passo.
- Confirmação de não implementação.
