# Arquitetura do HTML de Estudo CKOS

## Objetivo

Criar um HTML mobile-first, black background, com linguagem fácil, diagramas simples, exemplos práticos, exercícios e espaços visuais para prompts de imagem.

## O HTML não deve ser apenas texto

Ele deve ter:

- seções curtas;
- cards explicativos;
- glossário;
- diagramas simples;
- exercícios;
- flashcards estáticos;
- blocos “explique para founder”;
- blocos “explique para técnico”;
- blocos “erro comum”;
- espaços de imagem com wireframe;
- botão copiar prompt;
- commandbar visual simulada;
- animações CSS leves;
- motion suave;
- mapa do ecossistema;
- comparação com Apple, Apify, Manus, Claude, Codex e Branddock.

## Estrutura sugerida

1. Abertura: O que é CKOS em 1 minuto.
2. AI First de verdade: o que é e o que não é.
3. Tipos de AI First no mundo.
4. O mapa do sistema.
5. Runtime explicado.
6. Commandbar explicado.
7. Policy explicado.
8. Transformers explicado.
9. RAG, embeddings e memória.
10. Squads, superagents, agents e subagents.
11. Nodes, workflows e canvas.
12. Project Dashboard.
13. ROI.
14. Feedback.
15. Support.
16. Créditos e planos.
17. Research collectors.
18. Learning Mode.
19. CKStore.
20. Planner de projeto.
21. Como estudar o vault.
22. Exercícios.
23. Glossário.
24. Roadmap.

## Prompt spaces

Cada seção estratégica deve ter um espaço visual:

- wireframe placeholder;
- título da imagem;
- intenção visual;
- prompt curto;
- botão copiar.

## Atenção

Não gerar imagens agora. O HTML deve facilitar copiar prompt para o CK Visual Director gerar engenharia de prompt em outro chat.
