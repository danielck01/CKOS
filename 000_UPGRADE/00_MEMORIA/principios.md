# Princípios do Briefing AI FIRST

## 1. Universalidade
Todo pedido do usuário é uma intenção operacional. A intenção pode virar projeto, tarefa, proposta, roteiro, pesquisa, campanha, design, decisão, automação ou suporte.

## 2. Contexto antes de resposta
O sistema deve montar estado contextual antes de executar. O contexto pode vir do usuário, memória, RAG, documentos, conectores, Apify, web research, ferramentas internas e histórico de projeto.

## 3. Agentes antes de output
Outputs importantes devem nascer de agentes especializados, não de uma resposta única. O orquestrador decide quais agentes entram.

## 4. Ferramentas antes de achismo
Quando houver necessidade de dado externo, arquivo, scraping, cálculo, agenda, CRM, analytics, repositório ou busca, o sistema deve acionar ferramenta ou conector.

## 5. Aprovação para decisões sensíveis
Gastos, mudanças críticas, publicação, exclusão, proposta final, deploy, envio para cliente e alterações em ativos estratégicos exigem approval gate.

## 6. ROI desde o início
Cada briefing deve ter pelo menos uma hipótese de valor: economia de tempo, aumento de receita, redução de risco, ganho de clareza, aumento de conversão, velocidade de execução ou melhoria de qualidade.

## 7. Aprendizado contínuo
Todo briefing concluído deve registrar resultado, feedback, lições, padrões, erros e atualização de policies/skills quando necessário.
